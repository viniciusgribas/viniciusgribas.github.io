🇬🇧 - English

This Jupyter Notebook was developed to be a simple ETL process, Extracting data from a Web source, Transforming its null and non-usable values and Exporting them in .CSV format to a new output folder, where are added information about the last data updates as well as the variables already categorized according to their characteristics (String, Float and DateTime).

The developed ETL process composes the flow of a data engineering solution developed to provide Brazilian electric generation data in a dashboard type report, prioritizing to be iterative, dynamic and autonomous. Accessible by specialists in a timely manner.

For better understanding, the code was divided into three parts:

  • Data Extraction:
  • Using the requests library, a WebScrapping library that allows to request and extract data from the Brazilian Electric Energy Agency.
    • This same library allows you to check the latest updates made by the Host, as well as other features.
  • Data Transforms:
    • Here, the main library used was pandas, for manipulating DataFrames.
    • The missingno library, which specializes in missing values, allows you to view the null data and interpret its correlations.
  • Data Load
    • The os library was used to interact with the directories, creating a new folder for the output files.
    • The final Dataframe contains all the relevant variables structured in their correct typologies. It also has new columns, containing information about the date the output file was created and its last update by the National Electric Energy Agency

The detailed step-by-step is shown in the below.

Data Transformation¶

The following procedure will be adopted: 1) All variables will be classified according to their Type.

2) Unnecessary variables for the current analysis will be excluded by .drop() function.

3) The variables with null values will be analyzed individually.

 - The analysis will be done by `missingno` and `pandas` libraries.
 - Prioritizing not to reduce the dataframe, the missing values will be handled by the `.replace()` function.

4) After the analyses on null values, the variables will be renamed for better interpretation.

5) Finally, two new columns are created in Date-Time format. One containing information about the last update made by the host on data and the other containing information about the generation date this ETL output.

df.ETL_DataBase_LastModified = pd.to_datetime(df.ETL_DataBase_LastModified).dt.tz_localize(None)
